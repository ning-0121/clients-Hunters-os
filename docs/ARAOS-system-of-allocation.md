> **这是 ARAOS 的理论定位文档(定位层,不含实现)。**

## 1. 理论定位:SOE = System of Allocation

**SOE 不是 CRM、不是联系人系统、不是漏斗系统、不是数据系统。**

> **SOE = System of Allocation —— 一个资本配置/资源部署的决策系统。**
> 通俗名:**GTM Capital Allocation Engine(Go-To-Market 资本配置引擎)**。
> 它回答企业增长唯一的第一性问题:**"有限的增长资本,这一块该投到哪个未来现金流资产上?"**

**正式定义**:在**有限 GTM 资源约束**下,把销售资源**配置到最值得捕获的战略机会上**的决策系统。

**数学本体**:约束下的最优组合配置 / 投资组合构建 / 序贯决策(控制论)。**不属于**记录(Record)、数据(Data)、流程(Workflow)系统。

### 1.1 与巴菲特 / 桥水 / PE / 亚马逊 CapEx 同族

五个要素完全同构:**资产宇宙 · 有限预算 · 约束 · 目标函数 · 影子价格 + 动态/学习**。共同结构是:

> 在有限资本 + 风险/约束下,把资本配置到一组不确定、随时间变化的"未来回报资产"上,
> 最大化长期风险调整后回报;靠**影子价格(约束的边际机会成本)**决定每个仓位值不值;
> **动态再配置 + 结果回灌更新信念。**

### 1.2 但 SOE 是这一族里最难的变种(四点差异)

| # | 差异 | 含义 | 最像谁 |
|---|---|---|---|
| 1 | **资本不可逆、不可回收** | 花在死账户上的 rep 工时是沉没成本,卖不掉 | PE/VC、油气勘探 |
| 2 | **资产买不到、只能 origination** | 无法在市场买入"Gymshark 关系",只能用努力概率性原创出来;**获取成本本身随机(origination risk)** | 早期 VC |
| 3 | **预算是多维、不可替换的向量** | Capacity =〔工时/额度/差旅/老板精力/样品/现金风险〕互不可换 → 多约束 LP,每条一个影子价格 | 多约束运筹 |
| 4 | **回报对"动作+时机"内生** | 成交概率是动作和时机的函数(热窗口出手才转化) | 控制问题 / 非平稳多臂老虎机 / 实物期权 |

**结论**:SOE ≈ **"自营 origination 的 PE/VC 基金 + 实物期权择时 + 非平稳多臂老虎机 + 多维不可换预算"**——比巴菲特的流动公开股组合难得多。

## 2. 核心模型:Universe → Revenue 的十层闭环

```
1. UNIVERSE            全球所有潜在品牌(总账户母体)
   │  结构性筛选:工厂在不在这个局里(品类/地理/是否外包)
2. ADDRESSABLE UNIVERSE 值得一家工厂关注的品牌(有限、可枚举)= TAM
   │  把每个账户当资产定价
3. OPPORTUNITY SET     每账户 = 风险/origination 调整 NPV 资产(Value × Timing × Access ÷ Cost)
   │  按 Value 门槛策展 → 可投资宇宙
4. PORTFOLIO(Vault)   策展后的战略资产组合(在建 Opportunity + 收益期已赢账户)
   │  叠加企业 CAPACITY(多维预算 + 风险偏好)→ 影子价格
5. CAPACITY            企业 GTM 资源约束(多维不可换预算 + 风险偏好)
   │
6. ALLOCATION          约束下最优部署 GTM 资本 → 每资产一个 ACTION + 仓位
   │
7. ACTION              离散输出:Strike / Emergency Hunt / Nurture / Hold / Abandon
   │  执行(origination 努力)
8. DEAL / ORIGINATION  概率化索取权 → 合约现金流(行权 + 交割:概率坍缩、条款确定)
   │
9. REVENUE             已实现现金流(首单=origination 完成,复购=红利)
   │  结果回灌 → 校准 Opportunity(回报)+ 影子价格(容量)→ 下一轮更准
10. FEEDBACK LOOP /     情报复利 + 赢 logo 的组合外部性 + 学到的 ROIC 模型
    COMPOUNDING         → GTM 资本回报率随时间上升
```

**本质一句话**:Universe →(估值与策展)→ Opportunity 组合 →(用容量约束)→ Allocation →(origination)→ Revenue →(学习)→ 复利。一个"资本"是多维 GTM 资源、"资产"是被原创出来的账户关系的**资本配置闭环**。

### 2.1 最终理论模型:三层资本配置机器

```
L1 机会层(客观 / firm-invariant):账户 = 在建资产,风险/origination 调整 NPV
L2 容量层(主观 / opp-invariant):多维不可换预算 + 风险偏好 → 影子价格
L3 配置层(产品本体):argmax Σ 风险调整战略回报 s.t. 容量 → 每资产一个 ACTION
        ↑ 结果回灌校准 L1 回报 / L2 影子价格(复利)
```

- **L1 机会层**:机会属性是**客观的、与企业无关的**(Gymshark 的进口量、品类、时机,对任何工厂都一样)。
- **L2 容量层**:约束是**主观的、与机会无关的**(我方这周有多少 rep 工时、样品额度、现金风险偏好)。
- **L3 配置层**:把 L1 的资产价值与 L2 的影子价格相交,输出每个资产此刻该做什么动作。**这一层才是产品本体。**

> 此前把 Resource 当"第四轴"(塞进机会属性)是错的;机会(客观)与容量(企业)必须分层。

## 3. 术语定义(全系统统一)

| 术语 | 金融对应 | 定义 |
|---|---|---|
| **Account** | 标的实体 / 现金流发行方 | 真实世界那个品牌,独立于我们存在 —— **潜在未来现金流的发行方** |
| **Opportunity** | 资产 / 期权 | 我们持有的、**对某账户未来现金流的概率化、时间敏感的索取权**(在建资产);价值 = 风险/origination 调整后的 NPV。**不是线索,不是联系人。** |
| **Deal** | 行权 + 交割过程 | **Opportunity 的行权与交割过程**——把"概率化索取权"转成"合约现金流":不确定性在这里坍缩(P→1 或 0),条款在这里确定(margin/量) |
| **Revenue** | 已实现现金流 / 红利 | **已实现现金流**(真实订单与回款);首单 = 资产 origination 完成,复购 = 该战略资产的红利/收益流 |
| **Capacity** | 多维预算 + 风险偏好 | **企业 GTM 资源约束**:〔工时/额度/差旅/老板精力/样品/现金风险〕互不可换的向量 + 风险偏好 |
| **Shadow Price** | 约束的边际机会成本 | **某类资源在当前企业约束下的边际机会成本**——决定一个仓位"值不值得占用这份稀缺资源" |
| **Action** | 配置器的离散输出 | 配置器对每个资产输出的离散动作:**Strike · Emergency Hunt · Nurture · Hold · Abandon** |

**Action 五态**(L3 配置层的输出):

| Action | 含义 | 触发 |
|---|---|---|
| **Strike** | 出手行权——此刻是热窗口,集中资源转化 | 高价值 + 时机窗口开启 + 可达 |
| **Emergency Hunt** | 紧急 origination——价值高但缺接触路径,加速找人/建联 | 高价值 + Access 缺口 |
| **Nurture** | 持有并培育——保持低频投入,等待时机或情报成熟 | 中价值 / 时机未到 |
| **Hold** | 持有期权不行权——已建立、等待触发,不主动投入 | 已接触 / 等待外部信号 |
| **Abandon** | 平仓——价值不足或长期无回报,停止占用资本 | 低价值 / origination 反复失败 |

### 3.1 北极星与护栏

资本配置器的目标函数 = **最大化所投稀缺资本的长期风险调整后回报**。北极星必须是**回报率(ROIC)量纲**,不是绝对产出量纲。

| 指标 | 角色 | 说明 |
|---|---|---|
| **北极星 = GTM 资本的 ROIC** | ✓ 唯一北极星 | 单位 **binding 稀缺资源**投入,所捕获的**风险调整后、可复利**的战略价值。唯一带分母(稀缺资本)的回报率。 |
| 未来 12 月预期采购额 | "GTM AUM"(规模) | 组合的前瞻资产价值(分子),衡量规模,不衡量效率 |
| 高价值覆盖率 + Strike 赢率 | 质量护栏 | 健康度指标;100% 覆盖也可能 ROI 极差,故只作护栏 |
| 成交额 / Pipeline 额 | ✗ 不作北极星 | 产出/输入量纲,**没有分母(资本)**;大团队能低效堆出大成交额 |

> 类比基金:北极星不是"返还总额"、也不是"AUM",而是 **IRR / ROIC**。运营上:**用满预算(别闲置资本),在边际上最大化 ROIC。**

## 4. 为什么不是 CRM / Apollo / HubSpot

这些系统都在指认某个**本质功能**,而它们**全都在某个决策的下游**——都假设"你已经决定了追谁、投多少":

| 系统 | 本质 | 管什么 | 位置 |
|---|---|---|---|
| **Salesforce** | System of **Record** | 记录 / 记忆(state) | 决策下游 |
| **Apollo** | System of **Supply** | 供给 / 原料(data) | 决策下游 |
| **HubSpot** | System of **Workflow** | 流程 / 执行(process) | 决策下游 |
| **ARAOS / SOE** | **System of Allocation** | **配置 / 决策(allocation)** | **决策层本身** |

> SOE **消费** Apollo(供给)、**坐在** Salesforce(记录)之上、**驱动** HubSpot(流程),自己是**配置**。

**根本区别 —— 充裕世界 vs 稀缺世界**:
- CRM / Apollo / HubSpot 是**充裕世界**的工具(假设资源可得、决策已定,只管记录/供给/执行)。
- SOE 是**稀缺世界**的工具(从约束往外建,**决策本身就是产品**)。
- 对资源极度受限的外贸工厂,**约束才是主角**——这是**范畴差异,不是功能差异**。

**一句话**:Apollo 卖你"一个人";Salesforce 记你"做过什么";HubSpot 推你"流程走到哪";而 **SOE 是 go-to-market 的资本配置层**——它回答企业增长唯一的第一性问题:"有限的增长资本,这一块该投到哪个未来现金流资产上。"

## 5. 长期愿景:外贸 GTM 资本的 "Aladdin"

> ⚠️ **本节是未来三年的北极星,不是 P0。** 列在此处是为了让所有 P0 决策"朝同一个方向收敛",**不是要现在实现**。具体哪些现在不做,见 §6。

百亿美元级护城河**不是数据**(数据是商品,Apollo 更多)。一个资本配置器的持久优势 = **可持续的超额风险调整回报**,其不可复制性来自**三层复利**:

1. **专有的"决策→结果"数据飞轮**:每一次"对哪个机会花了什么 → 结果如何"的配置结果,是别人没有、且随使用增长的数据——**是决策结果的飞轮,不是联系人的飞轮**。
2. **校准的领域 ROIC 模型(alpha 模型)**:哪些信号在多大成本下预测多大订单结果——把数据变成**配置 edge**(Renaissance/桥水式护城河:不在于有数据,在于把数据变成更优配置决策、且越用越准)。
3. **组合外部性**:赢下标杆 logo → 相邻账户获取成本/风险下降(赢 Gymshark → Vuori 变便宜),模型学到这些协方差 → 组合回报**超线性**上升;若多租户,跨工厂的配置结果聚合成**全行业 ROIC 模型**,单一玩家造不出。

**终局形态**:**外贸 GTM 资本的 "Aladdin"(对标 BlackRock 的风险/配置操作系统)**——不是交易系统、不是数据源,而是**整个行业的增长资本部署决策都路由经过的"配置智能层"**:它比任何人都更懂"在这个市场,一块钱的 GTM 努力投到哪里回报最高",且这份认知随使用复利。

## 6. 短期 P0 如何保持收敛(关键纪律)

资本配置是**目标函数与语言**,不是现在就要造的发动机。当前阶段**绝不**为了"像配置器"而扩大 P0。

### 6.1 现在明确不做(全部后置到长期愿景)

- ❌ 完整 Allocation Engine(L3 的最优化求解器)
- ❌ 复杂影子价格(多约束 LP 的对偶解)
- ❌ 多维预算优化(多维不可换向量的运筹求解)
- ❌ Aladdin 版本(行业级配置智能层)
- ❌ 预测模型(校准的领域 ROIC / alpha 模型)
- ❌ 跨租户 ROIC 学习(组合外部性的全行业聚合)

### 6.2 现在做(优先级,严格按此顺序)

1. **Conversion OS P0 先上线** —— 已验收(本地 commit `40a1f0c`),应先推上线,让业务员能用**客户时间线 + Deal**。详见 `ARAOS-conversion-os.md`。
2. **Conversion OS P1** —— **Relationship Band · Risk Pool · Deal Board · Revenue Forecast**。详见 `ARAOS-conversion-os.md`。
3. **SAI / SOE P0** —— 只保留**轻量 Vault + Strategic Value + Access + Timing 的最小闭环**(不是完整 Allocation Engine)。详见 `ARAOS-sai-p0.md`。

### 6.3 概念映射:当前组件 ↔ 配置语言中的位置

> 这张表只用来**对齐语言、标定每个现有组件在最终模型里的位置**,**不改变任何当前实现、不新增字段、不新增能力**。

| 当前组件(已设计/已建) | 在配置模型中的位置 | 注 |
|---|---|---|
| Conversion OS · `deals` | **Deal = 行权 + 交割**(origination) | 概率坍缩 / 条款确定就发生在这里 |
| Conversion OS · `customer_events` 时间线 | **Feedback Loop 的原始记录**(决策→结果飞轮的底层) | 长期复利的数据来源 |
| Conversion OS · Relationship Band | **收益期资产的关系状态** | 已赢账户(红利流)的健康度 |
| Conversion OS · Revenue Forecast | **组合的前瞻 AUM(加权)** | 规模量纲,非北极星 |
| SAI · Prospect Vault | **Portfolio(在建资产组合)** | 当前以 `awaiting_contact` 复用实现 |
| SAI · Strategic Score | **资产的风险/origination 调整价值(Value 维度代理)** | 当前是纯函数,非完整 NPV 模型 |
| SAI · Contact Hunting | **Access 维度 + Emergency Hunt 动作** | origination 概率的提升手段 |
| SAI · Timing / 后续 Monitoring | **Timing 维度 + 实物期权窗口** | P0 仅最小化,Monitoring 是 P1+ |
| SAI · 频率档(daily/weekly/monthly/abandon) | **Action 的最小近似**(Strike/Nurture/Hold/Abandon 的雏形) | 完整 Action 引擎是长期 |

**一句话**:今天我们建的是配置器的**最小可用骨架**(看得懂资产、记得住结果、按价值分配耐心),**不是**配置器的最优化大脑。骨架先跑通、先产生业务价值、先积累"决策→结果"数据;**等数据证明 ROIC 再逐层加配置智能。**
